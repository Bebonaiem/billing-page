Introduction

BillingHub is a free, open-source billing and invoicing system built for hosting providers and service businesses. It is self-hosted, fully extensible, and ships with everything you need to accept payments, manage clients, and automate recurring billing.

Why BillingHub?

  • Self-hosted — your data stays on your server
  • No per-seat pricing — one deployment, unlimited staff
  • Extensible — build custom modules for any integration
  • Modern stack — PHP 8.3, Laravel, Vite, Tailwind
💡
TIP

Ready to install? Head straight to the Installation guide to get started.

Installing #

⭐
RECOMMENDED: Automated Installer

Use our automated installer script for a one-command setup. See Installation Methods below.

Option A: Automated Installation (Recommended)

The installer script handles everything automatically:

bash
# Clone the repository
mkdir -p /var/www
cd /var/www
git clone https://github.com/Bebonaiem/billing-page.git billinghub
cd billinghub/billing-system

# Run the automated installer
sudo bash scripts/install_ubuntu.sh

The script will prompt you for:

  • Application URL (e.g., https://your-domain.com or http://localhost)
  • Database name, username, and password
  • Admin account details (name, email, password)

The installer automatically configures Nginx, creates the database, runs migrations, and sets up the queue worker.

Option B: Manual Installation

If you prefer to set up manually, follow these steps:

Clone the repository

bash
mkdir -p /var/www
cd /var/www
git clone https://github.com/Bebonaiem/billing-page.git billinghub
cd billinghub/billing-system

Install dependencies

bash
composer install --no-dev --optimize-autoloader
npm install && npm run build

Set file permissions

bash
chmod -R 775 storage bootstrap/cache
chown -R www-data:www-data /var/www/billinghub/billing-system

Creating the Database #

📌
IMPORTANT

Replace yourPassword with a strong, unique password before running these commands.

bash
# Create database using sudo (handles auth_socket authentication)
sudo mysql -e "CREATE DATABASE billing_db;"
sudo mysql -e "CREATE USER 'billing_user'@'localhost' IDENTIFIED BY 'yourPassword';"
sudo mysql -e "GRANT ALL PRIVILEGES ON billing_db.* TO 'billing_user'@'localhost';"
sudo mysql -e "FLUSH PRIVILEGES;"

Setting Up .env #

Create the environment file

bash
cd /var/www/billinghub/billing-system

Generate encryption key

bash
php artisan key:generate --force
php artisan storage:link
🚨
DANGER

Back up your encryption key (APP_KEY in .env) immediately. It encrypts all sensitive data. If lost, data becomes irrecoverable.

Configure your database credentials

Open .env in your editor and update these values:

.env
APP_URL=https://your-domain.com

# IMPORTANT: Use 'localhost' (not 127.0.0.1) for Unix socket connection
DB_CONNECTION=mysql
DB_HOST=localhost
DB_PORT=3306
DB_DATABASE=billing_db
DB_USERNAME=billing_user
DB_PASSWORD=yourPassword

Setting Up the Database #

⚠️
WARNING

The migration command below may take a few minutes on first run. Do not interrupt the process.

bash
cd /var/www/billinghub/billing-system
php artisan migrate --force
php artisan db:seed --force

Optimize and cache

bash
php artisan optimize
php artisan view:cache

Create admin account

After installation, use the interactive command to create users or promote existing users to admin:

bash
cd /var/www/billinghub/billing-system
php artisan make:admin

The command will guide you through creating a new user or promoting an existing user:

Option 1: Create New User

The command will prompt for:

  • First name
  • Last name
  • Email address
  • Password (entered securely)
  • Role (admin or user)

Option 2: Promote Existing User

The command will ask for the user's email and let you change their role.

💡
TIP

You can create both admin and regular users with this command. Choose the appropriate role based on the user's access level.

📌
IMPORTANT

Only assign admin status to trusted users. Admin accounts have full system access including user management, billing, and system settings.

Cronjob & Queue Service #

Create the cronjob

Add a cron job to run Laravel's scheduler every minute:

cron
(crontab -l 2>/dev/null; echo "* * * * * cd /var/www/billinghub/billing-system && /usr/bin/php artisan schedule:run >> /dev/null 2>&1") | crontab -

Create the queue worker service

Create /etc/systemd/system/billing-system.service with the following content:

systemd
[Unit]
Description=Billing Hub Queue Worker
After=network.target mysql.service mariadb.service

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/var/www/billinghub/billing-system
ExecStart=/usr/bin/php /var/www/billinghub/billing-system/artisan queue:work --sleep=3 --tries=3 --queue=default,emails
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

Enable and start services

bash
systemctl daemon-reload
systemctl enable --now billing-system.service

BillingHub is now installed. Continue to Webserver Setup to configure Nginx and SSL.

Webserver Setup

Configure Nginx to serve BillingHub on your domain.

Nginx Configuration

Create a new site config at /etc/nginx/sites-available/billing-system:

nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/billinghub/billing-system/public;
    index index.php index.html;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}

Enable the site

bash
ln -sf /etc/nginx/sites-available/billing-system /etc/nginx/sites-enabled/billing-system
rm -f /etc/nginx/sites-enabled/default
nginx -t
systemctl reload nginx

CLI Reference

BillingHub exposes useful Artisan commands for maintenance tasks, user management, and diagnostics.

Admin Management

make:admin

Interactive user management command to create new users or promote existing users to administrator status.

bash
php artisan make:admin

Features:

  • Create new users with all required information
  • Promote existing users to admin or demote to regular user
  • Interactive prompts for user details (name, email, password, role)
  • Input validation and error handling
  • Secure password entry

Interactive Flow:

bash
=== User Management ===
What would you like to do?
❯ Create a new user
  Promote existing user to admin

=== Create New User ===
First name: John
Last name: Doe
Email address: john@example.com
Password: ****
Confirm password: ****
Select user role:
❯ Administrator (full access)
  Regular user (limited access)

✅ User created successfully!
Name: John Doe
Email: john@example.com
Role: Administrator
Status: active
�
TIP

This command can create both admin and regular users. Use it to quickly set up initial admin accounts or manage user roles without accessing the web interface.

Maintenance Commands

billing:generate-invoices

Generate recurring invoices for all active services.

bash
php artisan billing:generate-invoices

billing:add-late-fees

Add late fees to overdue invoices.

bash
php artisan billing:add-late-fees

billing:auto-suspend

Suspend services with overdue payments.

bash
php artisan billing:auto-suspend

Troubleshooting

Common fixes for installation and runtime issues.

MySQL Access Denied Error

⚠️
ERROR

SQLSTATE[HY000] [1698] Access denied for user 'root'@'localhost'

Cause: Ubuntu MySQL uses auth_socket authentication by default.

Solution: Use sudo mysql instead of password authentication, or set DB_HOST=localhost (not 127.0.0.1) in your .env file:

bash
# Create database with sudo
sudo mysql -e "CREATE DATABASE billing_db;"
sudo mysql -e "CREATE USER 'billing_user'@'localhost' IDENTIFIED BY 'password';"
sudo mysql -e "GRANT ALL PRIVILEGES ON billing_db.* TO 'billing_user'@'localhost';"

Node.js Version Too Old

⚠️
ERROR

Vite requires Node.js version 20.19+ or 22.12+

Solution: Install Node.js 20+ from NodeSource:

bash
# Install Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# Verify version
node -v  # Should show v20.x.x or higher

Permission Denied on Storage

⚠️
ERROR

Unable to write to storage directory

Solution: Set correct ownership and permissions:

bash
cd /var/www/billinghub/billing-system
chown -R www-data:www-data storage bootstrap/cache
chmod -R 775 storage bootstrap/cache

Nginx 502 Bad Gateway

Cause: PHP-FPM is not running or wrong socket path.

Solution:

bash
# Check PHP-FPM status
systemctl status php8.2-fpm

# Start if not running
systemctl start php8.2-fpm
systemctl enable php8.2-fpm

# Check socket path exists
ls -la /var/run/php/php8.2-fpm.sock

Database Migration Fails

If migrations fail, reset and try again:

bash
# Reset database
sudo mysql -e "DROP DATABASE billing_db; CREATE DATABASE billing_db;"

# Re-run migrations
cd /var/www/billinghub/billing-system
php artisan migrate:fresh --force
php artisan db:seed --force

Reset Everything and Start Fresh

To completely reset the installation:

bash
# Remove existing installation
cd /var/www
rm -rf billing-system

# Re-clone and reinstall
git clone https://github.com/Bebonaiem/billing-page.git billing-system
cd billing-system/billing-system
sudo bash scripts/install_ubuntu.sh
